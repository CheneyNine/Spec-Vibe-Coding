# Spec Vibe Coding

**有规划的 Vibe Coding，而不是一团乱麻。**

一个 [Claude Code](https://claude.ai/code) 技能（Skill），在你写下第一行代码之前，引导你完成专业的软件设计流程。先想清楚，再快速构建。

---

## 问题所在

AI 编程工具非常擅长写代码——但它们需要清晰的指令。大多数 Vibe Coding 项目失败，不是因为代码写得不好，而是因为从来没有人想清楚*要构建什么*、*如何组织它*，或者*用户体验究竟该是什么样的*。

Spec Vibe Coding 通过让设计阶段变得结构化、有引导、高效来解决这个问题。最终你会得到一套完整的设计文档，作为整个开发过程的事实来源。

---

## 它做什么

Spec Vibe Coding 在 Claude Code 内部以多阶段对话的形式运行。每个阶段都会问你正确的问题，生成可供修改的草稿，并将结果保存为项目目录下 `.spec-vibe/` 中的 Markdown 文件。

```
阶段 0：项目蓝图       → .spec-vibe/00-blueprint.md
阶段 1：需求文档 (PRD)  → .spec-vibe/01-requirements.md
阶段 2：系统设计        → .spec-vibe/02-system-design.md
阶段 3：UI/界面设计     → .spec-vibe/03-ui-design.md
阶段 4：开发任务        → .spec-vibe/04-dev-tasks.md
阶段 5：质量审查        → .spec-vibe/05-quality-report.md
```

最终你会得到：项目蓝图、产品需求文档、技术规格、界面规格，以及一套自包含的 AI 编程提示词——每个任务一个，可以直接粘贴到任何 AI 编程工具中使用。

---

## 适合谁用

- **非技术背景的创始人** — 有想法，但不知道如何向开发者描述清楚
- **初级开发者** — 跳过设计阶段，最终不得不把所有东西重写一遍
- **独立开发者** — 想快速推进，但总在做错误的功能
- **有经验的工程师** — 知道应该写规格文档，但就是不写

这个技能会根据你的技术水平自动调整语言和深度。你不需要事先知道什么是 PRD 或系统架构——Spec Vibe Coding 在实践中教会你。

## 安装

### 方式一：全局安装（推荐）

安装后对所有 Claude Code 项目生效。

```bash
# 克隆本仓库
git clone https://github.com/CheneyNine/Spec-Vibe-Coding.git

# 将技能文件复制到 Claude Code 全局技能目录
cp "Spec-Vibe-Coding/SKILL.md" ~/.claude/skills/spec-vibe-coding.md
```

### 方式二：项目级安装

仅对特定项目生效。

```bash
# 在项目根目录下执行
mkdir -p .claude/skills
cp "path/to/Spec-Vibe-Coding/SKILL.md" .claude/skills/spec-vibe-coding.md
```

> Claude Code 会自动读取 `~/.claude/skills/`（全局）和 `.claude/skills/`（项目级）目录下的技能文件。

## 使用方法

在 Claude Code 中自然地描述你的想法即可：

```
> 我想做一个帮助自由职业者管理发票的工具
```

```
> 我需要一个记录健身训练的 App
```

```
> 帮我规划一个多人文字游戏
```

Spec Vibe Coding 会自动触发，引导你完成每个设计阶段。你可以按顺序进行，也可以直接跳到任意阶段。

### 常用指令

| 你说 | 发生什么 |
|------|---------|
| `我想做...` | 启动项目蓝图阶段 |
| `需求` / `PRD` | 跳转到需求阶段 |
| `架构` / `技术选型` | 跳转到系统设计阶段 |
| `UI` / `界面设计` | 跳转到 UI/界面设计阶段 |
| `开始写代码` / `任务` | 生成开发任务提示词 |
| `质量检查` | 对所有文档进行一致性审查 |
| `跳过` / `下一步` | 跳到下一个阶段 |
| `进度` / `状态` | 查看各阶段整体进度 |

---

## 各阶段详解

### 阶段 0 — 项目蓝图
将模糊的想法转化为结构化的项目蓝图：问题陈述、目标用户、核心功能、技术方向、复杂度估算。

### 阶段 1 — 需求文档（PRD）
生成完整的产品需求文档，涵盖：用户画像、功能列表（含验收标准）、核心用户流程、约束条件、成功指标。

### 阶段 2 — 系统设计
将需求转化为技术架构：技术选型（含权衡分析）、组件架构图、数据模型、API 设计、第三方服务清单。

### 阶段 3 — UI/界面设计
**UI 项目：** 页面地图、用户流程、页面状态矩阵（正常/空数据/加载中/错误）、布局描述、视觉风格方向、组件清单。

**非 UI 项目（CLI、API、库）：** 接口清单、交互流程、输入/输出规格、状态矩阵、错误信息草稿。

### 阶段 4 — 开发任务
将项目拆解为 1–2 小时的任务单元，每个任务附带一个自包含的 AI 编程提示词，可直接粘贴到 Claude Code 或其他 AI 工具中。每个提示词包含：背景信息、前置工作、数据模型、API 规格、UI 规格、约束条件、验收标准和范围边界。

### 阶段 5 — 质量审查
交叉检查所有文档：覆盖度检查（PRD 中的每个功能是否都有对应的架构、UI 和任务）、一致性检查（字段名称在各文档中是否统一）、完整性检查（是否有缺失验收标准的功能）、范围检查（任务量对于你的时间线是否现实）。

## 输出文件结构

完成 Spec Vibe Coding 流程后，你的项目目录将包含：

```
your-project/
├── .spec-vibe/
│   ├── 00-blueprint.md        ← 项目蓝图
│   ├── 01-requirements.md     ← 需求文档（PRD）
│   ├── 02-system-design.md    ← 系统设计
│   ├── 03-ui-design.md        ← UI/界面设计
│   ├── 04-dev-tasks.md        ← 开发任务与 AI 提示词
│   └── 05-quality-report.md   ← 质量审查报告
├── src/
│   └── ...（你的代码）
└── ...
```

这些文件是你项目的设计事实来源。开发过程中遇到疑问（"这个功能具体该怎么做？"），随时回来查阅 `.spec-vibe/`。

## 核心理念

**先设计，再编码** — 几小时的结构化思考能节省几天的返工。设计文档让 AI 编程工具拥有做好工作所需的上下文。

**在实践中学习** — 你不需要事先懂得 PRD 或系统架构是什么。Spec Vibe Coding 会引导你走完整个流程，你会在实践中自然习得专业的软件设计思维。

**引导而非阻塞** — 你可以随时跳过任何阶段。质量审查阶段会捕捉所有重要的遗漏。进展比完美更重要。

**语言跟着你走** — 所有文档和回复都以你使用的语言生成。说中文，得到中文文档；说英文，得到英文文档。

**保持文档活跃** — 当你在项目中途新增功能时，Spec Vibe Coding 会主动提示哪些下游文档需要同步更新。
